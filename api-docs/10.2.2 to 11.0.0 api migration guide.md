## 1. Endpoint URL Changes

All renamed endpoints keep the same HTTP method and semantics. Only the URL path changed.

| Old URL | New URL |
|---|---|
| `GET /api/shops` | `GET /api/commodity/shops` |
| `GET /api/shops/{name}/transactions` | `GET /api/commodity/shops/{name}/transactions` |
| `GET /api/items` | `GET /api/commodity/items` |
| `GET /api/items/{name}/transactions` | `GET /api/commodity/items/{name}/transactions` |
| `GET /api/itemTypes` | `GET /api/commodity/item-types` |
| `GET /api/securityLevels` | `GET /api/security-levels` |
| `GET /api/locationTypes` | `GET /api/location-types` |
| `GET /api/crowdsource/locationBounties` | `GET /api/crowdsource/location-bounties` |

**Action required:** Update all HTTP client calls to use the new URLs.

## 2. Response Body Changes

### 2.1 `GET /api/commodity/items` (was `/api/items`)

**Old response** — array of plain strings:
```json
["Astatine", "Agricium", "Gold"]
```

**New response** — array of `CommodityItemDto` objects:
```json
[
  { "name": "Astatine" },
  { "name": "Agricium" },
  { "name": "Gold" }
]
```

**Action required:** Extract `.name` from each element instead of using the element directly.

---

### 2.2 `GET /api/commodity/item-types` (was `/api/itemTypes`)

**Old response** — array of plain strings:
```json
["Drug", "Food", "Medical"]
```

**New response** — array of `ItemTypeDto` objects with richer data:
```json
[
  { "name": "Drug", "displayName": "Drug", "parent": null },
  { "name": "Food", "displayName": "Food", "parent": null }
]
```

**Action required:** Extract `.name` (or use `.displayName` for UI labels). The `.parent` field is new and indicates a type hierarchy — `null` means top-level.

---

### 2.3 `GET /api/location-types` (was `/api/locationTypes`)

**Old response** — array of plain strings:
```json
["City", "Outpost", "Space Station"]
```

**New response** — array of `LocationTypeDto` objects:
```json
[
  { "name": "City" },
  { "name": "Outpost" },
  { "name": "Space Station" }
]
```

**Action required:** Extract `.name` from each element.

---

### 2.4 `GET /api/factions`

URL is **unchanged**, but the response type changed.

**Old response** — array of plain strings:
```json
["United Empire of Earth (UEE)", "Outlaw"]
```

**New response** — array of `FactionDto` objects:
```json
[
  { "name": "United Empire of Earth (UEE)" },
  { "name": "Outlaw" }
]
```

**Action required:** Extract `.name` from each element.

---

### 2.5 `GET /api/security-levels` (was `/api/securityLevels`)

URL changed **and** the `SecurityLevelDto` schema changed.

**Old `SecurityLevelDto`:**
```json
{ "id": 1, "name": "Low", "icon": "shield-low" }
```

**New `SecurityLevelDto`:**
```json
{ "level": 1, "name": "Low", "icon": "shield-low" }
```

**Action required:** Replace all references to `.id` with `.level`.

---

### 2.6 `GET /api/crowdsource/commodity-listings` — Pagination envelope changed

URL is **unchanged**, but the page wrapper schema was restructured.

**Old pagination envelope (`PageCrowdsourceCommodityListingsDto`):**
```json
{
  "content": [...],
  "totalElements": 42,
  "totalPages": 5,
  "size": 10,
  "number": 0,
  "numberOfElements": 10,
  "first": true,
  "last": false,
  "empty": false,
  "pageable": { ... },
  "sort": [ ... ]
}
```

**New pagination envelope (`PagedModelCrowdsourceCommodityListingsDto`):**
```json
{
  "content": [...],
  "page": {
    "size": 10,
    "number": 0,
    "totalElements": 42,
    "totalPages": 5
  }
}
```

**Action required:** Move pagination field access from the root level to the nested `page` object. Fields `numberOfElements`, `first`, `last`, `empty`, `pageable`, and `sort` are no longer present — remove any code that relied on them.

| Old field path | New field path |
|---|---|
| `.totalElements` | `.page.totalElements` |
| `.totalPages` | `.page.totalPages` |
| `.size` | `.page.size` |
| `.number` | `.page.number` |
| `.numberOfElements` | *(removed)* |
| `.first` | *(removed)* |
| `.last` | *(removed)* |
| `.empty` | *(removed — check `content.length === 0`)* |
| `.pageable` | *(removed)* |
| `.sort` | *(removed)* |

## 3. Request Body Changes (POST)

All POST endpoints that accepted form/filter DTOs have renamed fields. The old field names will cause a `400 Bad Request`.

### 3.1 Common fields renamed in all trade/filter DTOs

These fields were renamed in `TradeFormDto`, `BoundedTradeFormDto`, and `ItineraryFormDto`:

| Old field | New field |
|---|---|
| `factions` | `factionNames` |
| `factionsType` | `factionsNamesType` |
| `itemNames` | `commodityNames` |
| `itemNamesType` | `commodityNamesType` |
| `itemTypes` | `commodityTypes` |
| `itemTypesType` | `commodityTypesType` |

**Affected endpoints:**
- `POST /api/tools/trades`
- `POST /api/tools/circuits/{tradeId}`
- `POST /api/tools/itinerary`

### 3.2 `POST /api/tools/buyers` — `BuyerFormDto`

In addition to the common faction field renames above:

| Old field | New field |
|---|---|
| `factions` | `factionNames` |
| `factionsType` | `factionsNamesType` |
| `itemName` | `commodityName` |
| `itemQuantityInScu` | `commodityQuantityInScu` |

**Example — old request body:**
```json
{
  "commodityName": "Astatine",
  "commodityQuantityInScu": 100,
  "factionNames": ["United Empire of Earth (UEE)"],
  "factionsNamesType": "blacklist",
  "locationNamesType": "blacklist",
  "locationTypesType": "blacklist",
  "minSecurityLevel": 1,
  "supportedBoxSizeInScu": 32,
  "avoidHiddenLocations": false,
  "isCargoStolen": false,
  "allowWaitTimes": false
}
```

> **Note:** The example above already uses the **new** field names. Old clients sending `itemName`, `itemQuantityInScu`, or `factions` will receive a `400 Bad Request`.

## 4. New Endpoints

The following endpoints are new and have no equivalent in the old API.

### `GET /api/item/items`

Returns a paginated list of all items with full detail (`PagedModelItemItemDto`).

Each item includes:
```json
{ "name": "Pyro RYT Multi-Tool", "description": "...", "type": "Tool" }
```

### `GET /api/item/item-types`

Returns all item types as `ItemTypeDto[]`. Similar to `/api/commodity/item-types` but part of a richer item browsing API.

### `GET /api/item/item-types/{name}/items`

Returns a paginated list of items belonging to a given type (`PagedModelItemItemDto`).

### `GET /api/item/item-types/{name}/subtypes`

Returns child item types (`ItemTypeDto[]`) for a given parent type.

### `POST /api/tools/item-sellers`

Finds shops that sell a specific item. Accepts an `ItemSellerFormDto` body:

```json
{
  "itemName": "Pyro RYT Multi-Tool",
  "factionNames": [],
  "factionsNamesType": "blacklist",
  "locationNames": [],
  "locationNamesType": "blacklist",
  "locationTypes": [],
  "locationTypesType": "blacklist",
  "minSecurityLevel": 0,
  "supportedBoxSizeInScu": 32,
  "avoidHiddenLocations": false
}
```

Returns an array of `ItemTransactionDto`:
```json
[
  {
    "location": "Stanton > microTech",
    "shop": "Greycat Industrial",
    "securityLevel": 3,
    "faction": "United Empire of Earth (UEE)",
    "price": 245.50,
    "timestamp": "2026-03-31T00:00:00Z"
  }
]
```

## 6. Migration Checklist

Use this checklist to ensure your integration is fully updated:

**URL changes**
- [ ] Replace `/api/shops` → `/api/commodity/shops`
- [ ] Replace `/api/shops/{name}/transactions` → `/api/commodity/shops/{name}/transactions`
- [ ] Replace `/api/items` → `/api/commodity/items`
- [ ] Replace `/api/items/{name}/transactions` → `/api/commodity/items/{name}/transactions`
- [ ] Replace `/api/itemTypes` → `/api/commodity/item-types`
- [ ] Replace `/api/securityLevels` → `/api/security-levels`
- [ ] Replace `/api/locationTypes` → `/api/location-types`
- [ ] Replace `/api/crowdsource/locationBounties` → `/api/crowdsource/location-bounties`

**Response parsing**
- [ ] `/api/commodity/items` — extract `.name` from each `CommodityItemDto`
- [ ] `/api/commodity/item-types` — extract `.name` (or `.displayName`) from each `ItemTypeDto`
- [ ] `/api/location-types` — extract `.name` from each `LocationTypeDto`
- [ ] `/api/factions` — extract `.name` from each `FactionDto`
- [ ] `/api/security-levels` — rename all `.id` references to `.level` on `SecurityLevelDto`
- [ ] `/api/crowdsource/commodity-listings` — move pagination fields under `.page.*`
- [ ] Remove usage of removed pagination fields (`numberOfElements`, `first`, `last`, `empty`, `pageable`, `sort`)

**Request bodies (POST)**
- [ ] All trade DTOs: rename `factions` → `factionNames`
- [ ] All trade DTOs: rename `factionsType` → `factionsNamesType`
- [ ] `TradeFormDto`, `BoundedTradeFormDto`, `ItineraryFormDto`: rename `itemNames` → `commodityNames`
- [ ] `TradeFormDto`, `BoundedTradeFormDto`, `ItineraryFormDto`: rename `itemNamesType` → `commodityNamesType`
- [ ] `TradeFormDto`, `BoundedTradeFormDto`, `ItineraryFormDto`: rename `itemTypes` → `commodityTypes`
- [ ] `TradeFormDto`, `BoundedTradeFormDto`, `ItineraryFormDto`: rename `itemTypesType` → `commodityTypesType`
- [ ] `BuyerFormDto`: rename `itemName` → `commodityName`
- [ ] `BuyerFormDto`: rename `itemQuantityInScu` → `commodityQuantityInScu`
